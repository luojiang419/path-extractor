# 路径提取器

路径提取器是一款面向桌面用户的文件路径浏览与复制工具，基于 Flutter 构建，适合开发、设计、影视素材整理、办公归档等需要频繁定位文件、复制路径、整理路径清单的场景。

它不是传统意义上的重型文件管理器，而是围绕「快速找到文件、快速复制路径、快速整理一组路径」设计的轻量级生产力工具。你可以把它当作一个更顺手的路径提取面板：浏览本地磁盘、输入路径跳转、拖入文件集合、筛选目标文件，然后一键复制完整路径。

## 核心功能

### 快速浏览和复制路径

应用提供清晰的桌面文件浏览界面，支持从磁盘入口进入目录，也支持直接在路径输入框中粘贴本地路径并回车跳转。

在文件列表中：

- 点击文件会自动复制完整路径。
- 点击文件夹会进入该目录。
- 悬浮文件条目时可以点击复制按钮。
- 右键菜单支持复制路径、打开文件夹等操作。
- 复制成功后会显示提示，方便确认当前复制内容。

### 路径输入与面包屑导航

路径提取器支持通过顶部路径输入框快速跳转，也支持使用面包屑导航回到任意上级目录。

常见用法：

- 输入目录路径：直接进入目标目录。
- 输入文件路径：进入文件所在目录并高亮该文件。
- 输入无效路径：保留当前位置并提示路径无法访问。

### 拖拽整理与无限模式

把文件或文件夹拖入应用后，会进入「无限模式」。这个模式适合整理分散在不同目录里的文件集合。

在无限模式中，你可以：

- 拖入多个文件或文件夹作为根条目。
- 继续添加文件或文件夹。
- 浏览拖入的文件夹内容。
- 筛选、排序、复制路径。
- 从列表中移除不需要的条目。
- 一键清空当前集合。
- 按 `Esc` 退出无限模式。

这对素材整理、批量路径复制、项目资源归档非常实用。

### 筛选、排序和视图切换

应用支持按文件名筛选当前列表，并提供列表视图和网格视图。

排序能力包括：

- 按名称排序
- 按类型排序
- 升序或降序

目录会优先显示，方便在复杂目录中继续浏览。

### 文件类型识别与媒体缩略图

路径提取器会根据扩展名识别常见文件类型，并显示不同图标：

- 文件夹
- 图片
- 视频
- 音频
- 代码
- 文档
- 压缩包
- 其他文件

对于媒体文件：

- 图片文件直接显示缩略图。
- 视频文件可通过 `ffmpeg` 生成缓存缩略图。
- 如果缩略图生成失败，会自动回退为文件类型图标。

### 网络位置支持

在 Windows 环境下，应用支持添加和访问网络共享位置，例如：

- `\\server\share`
- `//server/share`
- `smb://server/share`

对于需要认证的共享目录，应用会提示输入用户名和密码，并尝试连接后打开目标位置。已添加的网络位置会出现在根视图中，方便后续快速访问或复制路径。

### 自动更新

Windows 客户端启动后会检查 GitHub Release 中的 `latest.json` 清单；如果发现新版本，会显示版本号、发布日期和更新说明。

更新流程支持：

- 手动检查更新
- 自动检查更新
- 下载更新安装包
- SHA256 校验
- 启动安装器并退出当前程序

## 使用场景

路径提取器适合这些工作流：

- 开发者复制文件、脚本、项目资源的完整路径
- 设计师整理图片、字体、源文件和导出资源
- 剪辑师整理视频、音频和素材文件
- 办公用户快速定位文档、压缩包和归档目录
- 将多处文件路径集中复制到表格、文档或聊天工具
- 访问 Windows 网络共享目录并快速提取 UNC 路径
- 从一批拖入文件中反复筛选、浏览和复制路径

## 快捷操作

- `Backspace`：非输入状态下返回上一级目录
- `Esc`：退出无限模式
- 单击文件：复制文件路径
- 单击文件夹：进入文件夹
- 右键文件条目：打开上下文菜单

## 本地开发

进入 Flutter 项目目录：

```bash
cd path_extractor_browser
flutter pub get
flutter run
```

运行测试：

```bash
flutter test
```

构建 Windows 版本：

```bash
flutter build windows --release
```

构建 macOS 版本：

```bash
flutter build macos --release
```

macOS 构建需要完整 Xcode。部分插件同时使用 CocoaPods 和 Swift Package Manager，首次构建前请确保 CocoaPods 可用。

## 发布流程

1. 更新 `path_extractor_browser/pubspec.yaml` 中的 `version`
2. 提交并推送 `main`
3. 创建并推送 `vX.Y.Z` tag
4. GitHub Actions 自动执行测试、构建、打包并发布 GitHub Release
